# Тесты Promptomptello

## 1. Unit-тесты (Vitest)

**Запуск:** `npm run test:unit` или `npm test`

### Файл: `src/test/unit/highlight.spec.ts`
Тестирует функцию `highlightText()` из `src/utils/highlight.ts`

| № | Тест | Что проверяет |
|---|---|---|
| 1 | `returns empty string for empty input` | Пустая строка на входе → пустая строка на выходе |
| 2 | `highlights XML tags` | `<div>hello</div>` → оборачивается в `span.token-xml-tag`, `<` экранируется в `&lt;` |
| 3 | `highlights variables` | `{{name}}` → оборачивается в `span.token-variable` |
| 4 | `highlights headers` | `## Title` → оборачивается в `span.token-header` |
| 5 | `escapes HTML characters` | `<b>test</b>` → `&lt;` и `&gt;` внутри span |

## 2. E2E-тесты (Playwright)

**Запуск:** `npm run test:e2e` (запускает `quasar dev` на порту 9000 автоматически)

### Файл: `src/test/e2e/app.spec.ts`

#### 1. Навигация по страницам
```
Главная → Каталог промптов → Шаблоны → База знаний → Поиск шаблона
```
Проверяет:
- URL изменяется при клике по пунктам сайдбара
- Заголовки страниц отображаются корректно

#### 2. Создание шаблона
```
Шаблоны → Добавить → Заполнить форму (название, категория, текст) → Создать
```
Проверяет:
- Редирект на `/templates` после создания
- Новый шаблон отображается в таблице

#### 3. Редактирование шаблона
```
Шаблоны → Редактировать → Изменить название → Сохранить
```
Проверяет:
- Форма заполняется данными существующего шаблона
- После сохранения — редирект на список
- Изменённое название видно в таблице

#### 4. Удаление шаблона
```
Шаблоны → Удалить → Подтвердить в диалоге
```
Проверяет:
- Диалог подтверждения отображается
- Уведомление об успешном удалении

#### 5. Auth guard (защита маршрутов)
```
/editor (без авторизации) → /login?redirect=%2Feditor → вход → /editor
```
Проверяет:
- Редирект на `/login` для защищённого маршрута
- Параметр `redirect` сохраняет исходный URL
- После входа — возврат на защищённый маршрут
